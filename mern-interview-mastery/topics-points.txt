Challenge 01: Debounce and Throttle
- File: js-challenges/01-debounce-throttle.js
- Line Count: 98 lines
- Key Line Placements:
  * debounce function: Lines 18-40
  * throttle function: Lines 51-71
  * demonstration test suite: Lines 77-98
- Core Interview Points:
  * Closures: Preserving the outer function scope variables (timeoutId, lastRan, lastFunc) between executions.
  * Context & Arguments Preservation: Using `func.apply(context, args)` to ensure `this` points to the correct executing scope and arguments are correctly forwarded.
  * Leading vs. Trailing Edge: Debounce immediate flag triggers the function on the leading edge (immediately) instead of the trailing edge. Throttle executes immediately on the first call, then buffers subsequent calls.
  * Browser Optimization: Crucial for optimization on paint-heavy events like scroll, resize, input search keyups to prevent thread blockage.

Challenge 02: Polyfills for Promise Combinators
- File: js-challenges/02-promise-polyfills.js
- Line Count: 156 lines
- Key Line Placements:
  * promiseAll: Lines 11-37
  * promiseAllSettled: Lines 43-71
  * promiseRace: Lines 78-88
  * promiseAny: Lines 94-119
- Core Interview Points:
  * Promise.all: Must return a promise; resolves only when all items resolve; rejects immediately if any item rejects. Needs to track completions via a counter rather than array length due to sparse arrays.
  * Promise.allSettled: Never rejects; maps all inputs to status objects: `{ status: 'fulfilled', value }` or `{ status: 'rejected', reason }`.
  * Promise.race: Attaches resolve/reject directly to the first settled input promise.
  * Promise.any: Resolves on the first success; if all fail, rejects with an AggregateError containing all error messages.
